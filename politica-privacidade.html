Privacidade e dados

Esta página explica como o protótipo utiliza informações localmente no navegador para simular carrinho, login, favoritos e formulários.

Nesta página

1. Dados utilizados 2. Finalidades 3. Armazenamento local 4. Segurança 5. Contato

1. Dados utilizados

O projeto pode guardar, apenas no navegador, informações digitadas em cadastro, login, favoritos, carrinho, checkout e formulário de contato.

Esses dados existem somente para tornar a navegação demonstrativa mais consistente.

2. Finalidades

Os dados locais permitem simular sessão de usuário, itens do carrinho, livros favoritos, endereço do checkout e mensagens enviadas visualmente pelo contato.

Nenhuma dessas informações é usada para autenticação real ou processamento externo.

3. Armazenamento local

A estrutura utiliza recursos como localStorage e sessionStorage para persistir estados do protótipo no próprio navegador.

Ao limpar os dados do site no navegador, carrinho, favoritos, sessão e formulários simulados podem desaparecer.

4. Segurança

Como o projeto é estático e demonstrativo, não há integração com banco de dados, gateway de pagamento ou servidor de e-mail.

Em uma loja real, essa política precisaria ser adaptada à legislação e aos serviços efetivamente utilizados.

5. Contato

Para fins de apresentação do site, dúvidas podem ser encaminhadas pela página de contato do próprio projeto.
